# M14.2 — Thronos AI Core body-contract alignment

This patch aligns `thronos_network` submit requests to the AI Core `/tx/submit` REST body contract shape.

## Request body contract

The submit request body is now:

```json
{
  "type": "AI_ATTESTATION",
  "payload": {
    "chain_id": "<chain_id>",
    "contract_address": "<contract_address>",
    "manifest_hash": "<manifest_hash>",
    "manifest_version": 1,
    "attestation_schema_version": "<schema_version>",
    "source_system": "<source_system>",
    "target_network": "<target_network>",
    "metadata": { "...": "..." }
  },
  "attestor_pubkey": "<attestor_pubkey>",
  "attestor_signature": "<attestor_signature>"
}
```

## Attestor fields

- `attestor_pubkey` and `attestor_signature` are required.
- `attestor_signature` can come from payload metadata (`metadata.attestor_signature`) or packed signer reference format:
  - `ATTESTATION_SIGNER_REF="<pubkey>::<signature>"`

## Response mapping

Accepted responses are normalized from REST fields:

- `tx_hash` from `tx_hash` or `txHash`
- `attestation_id` from `attestation_id` or `attestationId`
- `submission_id` from `submission_id` or `submissionId`

HTTP 400 maps to permanent malformed-request classification:

- `attestation_submit_bad_request`

## Unchanged

- generic RPC adapter path remains JSON-RPC-based
- receipt lifecycle/finality/replay fields remain contract-compatible
