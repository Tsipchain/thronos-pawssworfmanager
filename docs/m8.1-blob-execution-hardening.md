# M8.1 — Blob Execution Hardening

M8.1 hardens the first real blob execution path (`local_fs`) before any real attestation work.

## Blob hardening rules

- **Path safety:** blob ids are allowlisted and path resolution is enforced under configured root.
- **Atomic writes:** write uses temp file in root + `os.replace` for atomic swap behavior.
- **Overwrite semantics:**
  - `created` when blob does not exist,
  - `duplicate` when payload matches existing bytes,
  - `overwritten` when payload differs.
- **Size limit:** `MAX_BLOB_BYTES` (or constructor override) enforced before write.

## Error/receipt semantics

- Local fs adapter raises `BlobStorageError(code, failure_class, message)` for classified failures.
- Common codes:
  - `invalid_blob_id`
  - `path_escape`
  - `size_limit_exceeded`
  - `write_failed`
  - `read_not_found`
  - `delete_not_found`
  - `execution_gate_closed`
- Orchestrator maps blob write outcomes to `blob_receipt` with:
  - `status` (`not_configured|skipped_gate|created|duplicate|overwritten|failed`)
  - `failure_class`
  - `error_code`

## Guarantees

- If `execution_enabled=false`, no local fs write is attempted.
- Blob paths cannot escape configured root.
- Writes are size-checked and atomic-replace attempted.

## Non-guarantees

- no distributed locking/coordination
- no content-address dedupe across processes
- no durability/fsync guarantees beyond local host/filesystem behavior
- no blob encryption-at-rest runtime yet
- no real attestation submission yet
