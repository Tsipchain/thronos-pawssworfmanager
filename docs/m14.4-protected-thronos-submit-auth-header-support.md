# M14.4 — Protected Thronos submit auth-header support

This milestone documents optional auth-header support for non-AI-Core routes. For the AI Core submit route (`https://ai.thronoschain.org/tx/submit`), authorization is body-based (registered `attestor_pubkey` with `ai:attest` scope), not header-based.

Production target for this path is the AI Core endpoint above; explorer/api frontend routes are not the submit backend contract for this flow.

## Added provider config contract

- `ATTESTATION_AUTH_HEADER_NAME` (public)
- `ATTESTATION_AUTH_HEADER_REF` (sensitive ref, redacted in outputs)
- `ATTESTATION_AUTH_HEADER_PREFIX` (public, optional; e.g. `Bearer`)

If one of `name/ref` is set without the other, startup fails (`incomplete_attestation_auth_header_pair`).

Plaintext secret env input is forbidden:

- `ATTESTATION_AUTH_HEADER_VALUE`

## AI Core route note

For `https://ai.thronoschain.org/tx/submit`, the meaningful auth prerequisite is:

- attestor pubkey must be registered upstream
- registered scope must include `ai:attest`

The request body must include:

- `tx_type = AI_ATTESTATION`
- `attestor_pubkey`
- `attestor_signature`
- required payload shape

## Unchanged

- generic RPC path is unchanged
- receipt/error contracts remain stable
- upstream attestor registration + scope management remain external operational requirements
