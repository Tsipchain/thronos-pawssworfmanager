# M14.7 — AI Core upstream triage execution guide

Date: 2026-04-17 (UTC)

This guide turns the M14.6 checklist into repeatable commands/operators steps.

## 1) Run live health/routing probes

From this repository:

```bash
PYTHONPATH=src python scripts/run_ai_core_diagnostics.py --submit-url https://ai.thronoschain.org/tx/submit
```

Expected diagnosis categories (from current classifier contract):

- `upstream_service_suspended`
- `upstream_unavailable`
- `edge_method_forbidden`
- `registry_unregistered_service`
- `registry_missing_scope`
- `attestation_submitted`
- `unknown`

Current likely summary classes:

- `upstream_availability_incident`
- `upstream_gateway_method_policy_or_route_mapping`
- `registry_prerequisite_not_met`
- `submit_path_healthy`

## 2) Normalize attestor identity before registry checks

Use lowercase normalization consistently before any registry comparison or registration call.

Example:

```bash
PYTHONPATH=src python scripts/run_ai_core_diagnostics.py \
  --submit-url https://ai.thronoschain.org/tx/submit \
  --attestor-pubkey 0xABCDEF...
```

The output field `attestor_pubkey_lowercase` is the value to match against registry state.

## 3) Determine registry prerequisite status (upstream action)

This repository cannot directly verify AI registry presence without upstream registry access.

Operational requirement upstream:

1. confirm whether `attestor_pubkey_lowercase` already exists
2. if missing, run `AI_SERVICE_REGISTER` for that identity
3. ensure `ai:attest` scope is assigned

## 4) Decide next action using truthful buckets

- summary = `upstream_availability_incident`
  - classify as AI Core incident; do **not** change local request wiring
- summary = `upstream_gateway_method_policy_or_route_mapping`
  - fix route/method policy upstream; do **not** chase auth-header locally
- summary = `registry_prerequisite_not_met`
  - perform registration/scope steps upstream
- summary = `submit_path_healthy`
  - proceed with real attestation receipt verification/polling
- summary = `needs_manual_upstream_triage`
  - capture raw response and verify upstream contract owners before local code changes

## 5) Guardrail

Do not re-open explorer/api-domain confusion and do not pursue speculative header-auth rewrites for `https://ai.thronoschain.org/tx/submit` unless upstream contract owners provide exact mismatch evidence.
