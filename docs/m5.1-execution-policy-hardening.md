# M5.1 — Execution Policy Hardening

This milestone hardens startup policy so forbidden backend/mode combinations cannot boot.

## Explicit matrix

### Blob backend + execution mode
- `in_memory + dry_run` => allowed
- `in_memory + execute` => allowed
- `real-like (s3/gcs/azure_blob) + dry_run` => allowed
- `real-like (s3/gcs/azure_blob) + execute` => forbidden

### Attestation backend + execution mode
- `fake + dry_run` => allowed
- `fake + execute` => allowed
- `chain-like (thronos_chain) + dry_run` => allowed
- `chain-like (thronos_chain) + execute` => forbidden

## Startup refusal rules

Startup is fail-closed:
- Unknown backend or mode values are rejected.
- Any forbidden matrix pair is rejected with `forbidden_backend_execution_combination`.

## Capability/reporting consistency

Runtime capability output now includes:
- selected adapter backends and execution mode
- computed execution policy status
- `startup_allowed` and enforcement mode (`fail_closed`)

Metadata also includes `execution_policy_enforced=true` when startup passes.

## Remaining blockers before real execution

- real storage client integrations
- real chain submission/signing path
- secrets management
- auth and encryption runtimes
- DB/distributed coordination
